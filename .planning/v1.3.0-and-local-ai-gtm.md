## Local-AI GTM

### Positioning thesis
Local AI is moving from "I'll try a 7B once" to "I'll run a 70B for 11 hours
doing real work." That changes what users want from supporting tools. They
don't want another model runner — they want **the thing that stops their
machine from OOM-killing the model halfway through.**

DevPulse is *not* itself an offline-AI tool. It's the **co-pilot for tools
that are**. Copy direction: anti-cloud, pro-shipping, anti-babysitting,
pro-script.

### Audience overlap with TUI/CLI ethos
Per <https://wiki.alcidesfonseca.com/blog/why-tuis-are-back/> — the cohort
returning to keyboard-driven tools is exactly DevPulse's ICP. They will skip
"yet another menu bar app." The CLI surface is what makes us legible to them.
The menu bar is the gateway drug; the CLI is the retention.

### Don't try to rank for "offline AI" with one page
That keyword is occupied by Ollama / LM Studio / LocalAI. Topical-cluster
strategy under `/local-ai/`:

#### Cluster pages to build (priority order)
- [x] `/local-ai` — hub page (live in PR #6)
- [ ] `/local-ai/ollama-out-of-memory-mac` — high-intent pain query, we have
      all the content from `--before-load` + `--auto-clean`
- [ ] `/local-ai/run-70b-llama-on-macbook` — the 11-hour flight setup, written
      as a complete how-to (linkable on HN/Reddit)
- [ ] `/local-ai/free-vram-before-loading-model` — generic across backends
- [ ] `/local-ai/airplane-mode-llm-workflow` — the IG meme as a guide
- [ ] `/local-ai/lm-studio-vs-ollama-vs-llama-cpp-memory` — comparison,
      links to `can-i-run/<model>` pages
- [ ] `/local-ai/quantization-guide-mac` — Q4_K_M / Q8_0 / IQ2 in plain
      English, what fits on what spec

#### Cross-linking
- [ ] Every `can-i-run/<model>` page → "running this model offline →" link
      to `/local-ai`.
- [ ] Footer link to `/local-ai` everywhere.
- [ ] Homepage CLI section CTA already → `/local-ai`.

#### Schema markup
- [ ] `HowTo` schema on `/local-ai` (steps for pre-flight + babysit).
- [ ] `FAQ` schema for the obvious questions (do I need internet, how much
      RAM for 70B, can a MacBook do this, etc.).
- [ ] Validate with Google Rich Results test before push.

#### Distribution / backlinks
- [ ] **Submit `/local-ai/run-70b-llama-on-macbook` to HN** — as a *tutorial*,
      not a launch. ICP-friendly headline: "Running 70B Llama offline on a
      MacBook for 11 hours — the setup."
- [ ] **r/LocalLLaMA post.** Same content, different framing: practical setup
      + open-source. Avoid sounding like a product pitch.
- [ ] **GitHub issue replies.** Skim "ollama out of memory", "llama.cpp OOM"
      issues; reply with a concrete answer + link to the relevant cluster
      page. Highly leveraged; permanent backlinks.
- [ ] **Awesome-Ollama / Awesome-LocalAI list inclusion.** PR ourselves.

#### Content for AI agents that cite
- LLM-cited content (Perplexity, Claude, ChatGPT search) tends to lean on
  structured information: exit codes, JSON schemas, bullet steps. The
  `/local-ai` page already does this. Apply the same pattern across the
  cluster.
